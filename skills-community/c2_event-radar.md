# c2 週報——上周兑现回顾+本周事件预告

本 skill 仅供社群运营人员触发使用（群成员不直接交互），周日或周一固定跑一次。产出是单篇"双段周报"：正文分两节——上半场"本週回顧"核对上周预告事件的兑现情况，下半场"本週看點"预告本周新出现的财报与宏观事件及其影响标的。两节共用同一篇贴文，标题、一句话先懂、名词卡、免责声明只在贴文最外层各出现一次（见第 4 节产出模板）。

## 1. 双段结构：上半场·本週回顧 + 下半场·本週看點

### 上半场·本週回顧（零新增调用）

素材来源：上周预告贴（对话内引用，或运营手动粘贴）+ 本周 c1 早报已经拉过的数据。**零新增调用**——只有兑现数据出现缺口时才补拉一次快照，最多 ≤1 点额度。这是全 bundle 教育价值最高的板块：教新手看懂"预期 vs 兑现"的落差，比单纯预告更有教育意义。

逐条核对格式（上周预告过的每个事件各一行）：

`事件 → 当时预告 → 实际结果 → 一句白话解读`

示例：`GM 财报 → 上周预告"市场在赌全年指引上修" → 实际上调全年指引，盘中 +3.6% → 白话：这次赌对了，公司真的兑现了预期`

财报类逐条核对写 beat/miss 一句话；宏观类逐条核对写实际值 vs 预期值一句话。

### 下半场·本週看點（≈3/周）

即原有的事件预告设计：整理未来 7 天的重点财报与宏观数据发布，提示每个事件市场在赌什么、会影响哪些标的。调用序列（设计文档对应章节）：

| 步骤 | 调用 | 额度 |
|---|---|---|
| 1 | `metrics(query="earnings calendar", asset_type="tradfi", date_from/date_to=未来7天)` | 1 |
| 2 | 初筛 shortlist 分批快照补市值（每批 ≤5） | 1-2 |
| 3 | `metrics(query="economic calendar", categories=["macro"], country="US", date_from/date_to=未来7天)` | 1 |

三步各自的要点：
- **步骤 1**：财报日历，query+date 窗口拿未来 7 天的美股财报名单；返回全球交易所混排、无市值字段，过滤链见第 2 节（N-2）。
- **步骤 2**：对步骤 1 初筛后的 shortlist 分批补市值（每批最多 5 只），用于最终按市值排序取 Top 5-8。
- **步骤 3**：经济日历。使用 `categories=["macro"]`、`country="US"` 和明确 `date_from/date_to`。2026-07-23 在线回归确认单次调用可返回多日窗口。仍需按 `impact` 筛选，并对 CPI、FOMC、非农等高影响事件做日期核实。

调用形态铁律（本序列全程通用）：

> **当前调用形态（2026-07-23 回归）**：按 tool schema 直接传 `keywords/categories/sources` 数组；`metrics` / `signal` 每次最多 5 个 keywords，超出分批，并检查 `meta.warnings` 与 `meta.filters_applied`。若特定客户端仍报 -32602，再降级为单 ticker 调用；不要把多个 ticker 拼进 query。

每步 query 主形态调用示例：

```
1. metrics(query="earnings calendar", asset_type="tradfi", date_from="<今日>", date_to="<未来第7天>")
   # 财报日历；返回全球交易所混排、无市值字段，过滤规则见第 2 节（N-2）

2. metrics(keywords=["<TICKER1>","<TICKER2>"], query="live market snapshot", categories=["market"], asset_type="tradfi")
   # 初筛 shortlist 分批补市值；market 每次最多 5 keywords，超过需分批

3. metrics(query="economic calendar", categories=["macro"], country="US", date_from="<今日>", date_to="<未来第7天>")
   # 单次多日窗口；按 impact 筛选并交叉核实高影响事件日期
```

## 2. 财报日历过滤规则（N-2）

第 1 节步骤 1 拿到的财报日历，返回是全球交易所混排、且不带市值字段，必须过滤才能用。过滤链：

> 财报日历过滤（新 caveat N-2）：只留无交易所后缀的美股 symbol → `revenueEstimated > $1B` 初筛 → 市值排序取 Top 5-8。

对应 caveats SSOT 的 N-2 原文：

> N-2：earnings calendar 市场级可用（query+date_from/to），但返回全球交易所混排、无市值字段；过滤 = 无后缀 symbol + revenueEstimated 初筛 + 二次调用补市值。

具体执行三步：
1. **无交易所后缀 symbol**：只留形如 `AAPL` 的美股 symbol，剔除带 `.KS`/`.HK`/`.T` 等后缀的非美股条目。
2. **revenueEstimated > $1B 初筛**：用返回字段自带的 `revenueEstimated` 过滤掉营收规模太小、新手社群不关心的标的。
3. **市值排序取 Top 5-8**：初筛后的 shortlist（≤10）走第 1 节步骤 2 批量补市值，按市值从高到低排序，最终成稿只保留 Top 5-8 只，避免贴文塞满冷门标的。

三步缺一都可能把非美股、冷门小票塞进贴文——这条过滤链是本节唯一职责，和第 1 节步骤 3 的经济日历过滤（红线 10）是两套独立规则，不要混用。

## 3. 日期核实制度

> **日期核实制度（借鉴 trend-scout 弃用 fmp 日历的前科）**：trend-scout 连续 3 周实测 fmp 财报/经济日历返回垃圾（外币小票、关键词误解析成 ticker），最终弃用改为人工核实的事件锚点登记表。c2 保留日历调用作候选源，但加两道闸：① 进贴文的重点财报日期，用 `news(query="<公司名> earnings date")` 交叉确认（0 额度）或对照公司 IR；② CPI/非农等宏观日期禁按惯例推算，只认日历返回值并在贴文标注"以官方公告為準"。c2 每周产出的预告贴本身即社群版锚点表——下周跑时先对照上周预告核对差异，日期变更主动发更正。

实操拆成三道闸：

① **财报日期闸**：进贴文的每个重点财报日期，用 `news(query="<公司名> earnings date")` 交叉确认，0 额度；或对照公司 IR 页面核实。

② **宏观日期闸**：CPI、非农等宏观数据发布日期禁止按惯例推算——"CPI 通常在月中公布"这类经验判断不算数，trend-scout 就是被 CPI 日期临时变更坑过 3 天。只认日历调用返回值本身，并在贴文里标注"以官方公告為準"。呼应 S-7 铁律4：

> 铁律4 时间：宏观/财报时间标 美东+台北 双时区；禁按惯例推算日期。

③ **锚点表闸**：c2 每周产出的预告贴，本身就是社群版的事件锚点登记表。下周跑 c2 之前，先把这周产出的预告贴和上周的预告贴逐条核对——如果发现日期变了（比如原定财报日期推迟、宏观数据发布时间调整），必须主动发更正贴，禁止悄悄换个日期就带过去。更正贴用 S-8 模板（逐字）：

> 「🔁 更正｜MM/DD 那篇〈标题〉
> 錯在哪：…
> 正確是：…
> 原因：…（一句）
> 造成困擾抱歉，我們對數據負責。」

24h 内主动发；c2 在下周核对时一旦发现矛盾，必须提示运营，禁止静默带过。

## 4. 产出模板

贴文骨架沿用 S-3 七段式，但去掉第 7 段互动钩子——S-3 规定互动钩子只出现在 c1/c5，本 skill 是"回顾+预告"的复盘性质内容，不适合加开放式讨论钩子。

设计文档对应章节）：

> 产出结构：本周财报（谁、哪天、市场在赌什么）→ 本周宏观数据（哪天、为什么重要）→ 每个事件一句"影响哪些标的" → 名词卡。

六段骨架：
1. **标题**：📌 + 週報 + 日期区间
2. **一句話先懂**（≤40 字，概括本周最重要的一条兑现结果或最重要的一个预告事件）
3. **模块正文**，分两节：
   - **上半场·本週回顧**：逐条 `事件 → 当时预告 → 实际结果 → 一句白话解读`
   - **下半场·本週看點**：每个事件一行，格式固定为「谁｜哪天(美东+台北)｜市场在赌什么｜影响标的」
4. **接下來看什麼**：提示"下週回顧會逐一核對這些事件的兌現結果"，呼应第 3 节的锚点表机制
5. **今日名詞卡**：从本周回顾或本周看点内容里抽一个术语
6. 免责声明（S-11 对外版，逐字）："⚠️ 以上整理自公開研報與市場數據，僅做資訊分享，不構成投資建議。"

字数按 S-4 默认档：单篇 500-800 字（回顾段+预告段合计——S-4 没有把 c2 单列例外，两段内容要克制，逐条尽量一句话说完）。

以下是结构示范样例（非实测数据，实现时按当次调用返回替换；排版规则不变——纯文字＋emoji，不套 markdown 加粗/标题/表格，层级靠 emoji ＋ 空行 ＋「｜」——S-9 镜像）：

```
📌 週報｜7/20–7/26

一句話先懂：上週預告的 GM 財報兌現了「上修全年指引」的市場預期，這週輪到輝達生態鏈財報季開打。

本週回顧
1️⃣ GM 財報 → 上週預告：市場在賭全年指引上修 → 實際：上調全年指引，盤中 +3.6% → 白話：這次賭對了，公司真的兌現了預期
2️⃣ CPI（上週四公布）→ 上週預告：市場預期年增 3.1% → 實際：3.0%，小幅低於預期 → 白話：通膨降溫比想像中快一點，市場鬆一口氣

本週看點
🏢 NVDA｜7/24（週四）美東盤後／台北隔日清晨｜市場在賭AI晶片供給缺口能不能撐住營收指引｜影響標的：NVDA、MU、SK海力士
🏢 TSLA｜7/23（週三）美東盤後／台北隔日清晨｜市場在賭交付量疲弱會不會被毛利率數字蓋過去｜影響標的：TSLA
📊 非農就業（7/25 週五）｜美東 08:30／台北 20:30｜為什麼重要：決定聯準會下次會議降不降息｜以官方公告為準

接下來看什麼：下週回顧會逐一核對這些事件的兌現結果，日期若有變動會另外發更正。

今日名詞：財報指引
公司自己對下一季或全年營收、獲利的預測。可以粗略理解成：公司給市場的「劇本」，市場會拿實際結果去對照這個劇本有沒有兌現。

⚠️ 以上整理自公開研報與市場數據，僅做資訊分享，不構成投資建議。
```

## 5. 防坑镜像 + 发前自检 + 额度哨兵

防坑镜像（逐条，来源编号见括号）：

- **财报日历过滤链**（N-2，见第 2 节）：无后缀 symbol → revenueEstimated > $1B 初筛 → 市值排序取 Top 5-8，缺一步都可能把非美股或冷门标的塞进贴文。
- **经济日历"本周"禁令**（红线 10，见第 1 节）：query 绝对不能带"本周"，只能读日历返回值本身的日期字段；"本周"被解析成 lookback 7 天，返回的是已发布历史而非前瞻日历。
- **数组参数**（N-8 已关闭）：2026-07-23 当前 schema 在线回归确认数组可用；特定客户端仍报 -32602 时才降级为单 ticker 调用。
- **shortlist 批量上限**（B-31 镜像）：初筛后批量补市值每次最多 5 个 keywords，超过分批。
- **双时区+禁惯例推算**（S-7 铁律4，见第 3 节）：宏观/财报时间一律标 美东+台北 双时区，禁止凭经验推算日期。
- **SSE 并发上限**（红线 2）：任何拆成多路单查的步骤，单批 ≤4 路并发，超了 session 可能挂。

发前自检（S-12，逐字，每篇产出前逐项过）：

> 繁体✓ 大白话✓ 字数✓ 名词卡✓ 免责✓ 多空平衡✓ 单源标注✓ 价格可回溯✓

c2 专属追加一项：**日期可回溯✓**——每个财报/宏观日期都能指向当次调用返回值或 news 交叉确认结果，不是凭记忆或惯例写的（呼应第 3 节日期核实制度）。

额度哨兵（内部提醒，不进对外贴文，设计文档对应章节）：

> **额度哨兵**：每次模块跑完读一次 `meta.quota`（每个返回都带，实测），`remaining/limit < 15%` 时在产出末尾附一行内部提醒（不进对外贴文）："⚠️ 本月額度剩 N 次，按當前節奏約可跑 X 天，考慮降頻或升級"。额度见底导致日更断档比任何单篇内容缺失都伤运营。
