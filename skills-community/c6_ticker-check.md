# c6 标的速查——运营内部选题工具，产出不对外（随时触发）

本 skill 仅供社群运营人员触发使用（群成员不直接交互，也不会看到本 skill 的任何产出）。spec 逐字定位：

> 定位：**运营内部工具，产出不对外**。两个场景：① 运营看到群里在热聊某个票，想判断"值不值得为它出内容"；② 写贴文前快速摸底一个标的。输出是**内部备忘样式**（数据密度优先，不做贴文排版、不写科普白话），并以明确的出内容建议收尾。

这是本 skill 和 c1-c5 最关键的区别，必须写在最前面：c1-c5 的产出是"运营零加工即发"的社群贴文，c6 的产出是"运营自己看"的内部备忘草稿——**不套用 S-3 贴文骨架**，没有标题+一句话先懂的对外排版、没有今日名词卡（S-10）、没有对外版免责声明（S-11）、没有互动钩子；备忘只在文末固定加一行内部专属提示（S-11 内部版，见第 2 节），提醒运营这份东西不能直接复制进群。触发本 skill 不会产出任何可以直接贴出去的内容，只产出一份判断依据加一句明确立场。

两个场景走同一份调用序列与备忘模板，不分叉：场景①是运营看到群里在热聊某标的，想知道值不值得追；场景②是运营准备写 c1/c3/c4/c5 任一模块前，先摸底一个标的的基本面再决定要不要写。

## 1. 调用序列（≈2 点/次）

调用序列：

| 步骤 | 调用 | 额度 |
|---|---|---|
| 1 | `metrics(query="<TICKER> price target analyst ratings", asset_type="tradfi")` 快照 + 分析师共识（中位/最高/最低目标价） | 1 |
| 2 | `news(query="<TICKER> <公司名>", time_range="7d", limit=5)` 近期新闻要点 | 0（实测） |
| 3 | （可选，热议标的才加）`signal(query="<TICKER> 详细仓位", …)` 内部人/喊单/实盘温度 | 1 |

步骤 3 只在标的本身是"热议标的"时才加跑——比如触发本次速查正是因为群里在热议它（场景①），或步骤 1-2 的数据已经显示情绪面/新闻热度明显。单纯写贴文前的常规摸底（场景②）多数情况下两步（≈1 点）就够，不必每次都跑步骤 3。

调用形态铁律（架构 §2 镜像，本序列全程通用，即 N-8 登记项的操作化版本）：

> **调用形态铁律（2026-07-20 起生效，trend-scout v1.11.1 实测 + 2026-07-22 复现）**：`keywords/categories/sources` 等数组参数在当前环境会被序列化成字符串遭 schema 拒（连环 -32602）。所有调用规范以 **query 自然语言/空格拼串为主写法**（服务端自解析成 keywords，meta 可验证），数组形式仅作"标准客户端若可传数组"的备选注记。批量降级梯：① keywords 数组批量（≤10，B-31）→ ② query 串批量（crypto 实测可行，tradfi 多 ticker 可能被路由到 fundamentals，实现时验证）→ ③ 单 ticker 并行、每批 ≤4 路（SSE 红线）。另：Followin session 每 5-8 次调用可能短挂，重试 1 次即恢复，还不行让运营 `/mcp restart followin`。

每步 query 主形态调用示例：

```
1. metrics(query="<TICKER> price target analyst ratings", asset_type="tradfi")
   # 快照+分析师共识一次拿全：中位/最高/最低目标价、参与家数；行情位置段落的"现价对 52 周区间"同样取自这次快照，不必另外调用
   # 若 ticker 非美股正股（代币化/OTC/仙股/加密），本步骤按第 3 节防坑规则直接短路，不必往下跑步骤 2-3

2. news(query="<TICKER> <公司名>", time_range="7d", limit=5)
   # 搜索模式：不传 asset_type（红线 1）；quota=0（实测）；limit=5 控制返回条数，近期叙事段落只需挑 1 条最具代表性的

3. signal(query="<TICKER> 详细仓位", asset_type="tradfi", time_range="7d")
   # 可选，仅热议标的才加：内部人/喊单/实盘四维钻取，做法同 c4 步骤 2、c5 步骤 5；query 禁放"KOL"等元词——会被解析成 crypto 关键词，实测空返（防坑镜像见第 3 节）
```

## 2. 备忘模板：内部速查（≤300 字）

> 内部备忘固定结构（≤300 字）：**行情位置**（现价对 52 周区间）→ **机构共识**（家数+中位+分歧幅度）→ **近期叙事**（新闻/推特一条）→ **信号面**（有钻取时）→ **出内容建议**：🔥 值得速报（衔接 c5 直接成稿）/ 📌 并入下期温度计或研究笔记 / 🚫 不值得写 + 一句原因（无叙事层/纯情绪/陈货）。

落地结构，五段固定顺序，不套 S-3 贴文骨架：

1. **行情位置**：现价 + 相对 52 周区间的位置（上緣/中段/下緣），取自步骤 1 快照，不额外调用。
2. **机构共识**：参与家数 + 目标价中位数 + 分歧幅度（最高价–最低价区间）。**禁止只给单一均值**——这条规则不是 c6 独有，是约束 bundle 内所有对外模块的总则，c3 機構怎麼看段落"目标价必带家数+分歧幅度"是同一条规则的对外版（见 c3 第 2 节，原文标注"约束所有对外模块的总则，c6 同条镜像"），c6 只是率先在内部备忘里执行。
3. **近期叙事**：从步骤 2 的新闻里挑最具代表性的一条，一句话说完是什么、为什么最近被讨论。
4. **信号面**（可选，只在跑了步骤 3 时出现）：内部人/喊单/实盘任一维度有明显读数就写一句；没有钻取步骤 3 就整段省略，不写"无数据"占位。
5. **出内容建议**（收尾，三选一，必须表态，不能只列数据不给立场）：
   - **🔥 值得速报**：叙事新鲜（近期叙事是 7 天内新料，不是老调重弹）+ 标的明确 + 机构共识或信号面有明显异动（分歧幅度罕见收窄成一致、或信号面出现强烈单边读数）三者叠加才够格。备忘写出"🔥 值得速报"后即衔接 c5：运营可直接说"就 XX 出一篇"，跳过 c5 第一步扫描直接进第二步成稿（c5 正文已注明"运营是从 c6 标的速查的'🔥 值得速报'建议衔接过来"这条路由；若该标的当日恰好有财报，c5 会进一步路由到财报速读子型）。
   - **📌 并入下期温度计或研究笔记**：叙事存在但不够急迫，不必单独抢出一篇——够格进 c4 温度计下一轮的喊单榜候选（情绪/喊单热度够但没有独立速报的催化事件），或 c3 研究笔记的候选标的（机构共识扎实但没有"现在必须写"的紧迫性）。备忘必须写明具体并入 c4 还是 c3，不能只写"并入"不说去哪。
   - **🚫 不值得写** + 一句原因：三种常见原因——**无叙事层**（近期叙事段落挑不出像样的新闻/推特由头，纯粹是数字波动）、**纯情绪**（推特/喊单热但机构共识与信号面都没有实质印证，情绪本身撑不起一篇内容）、**陈货**（近期叙事段落唯一能找到的素材是超过 7 天的旧新闻，没有新鲜由头）。三选一写一句说明具体命中哪种。

判断责任句（同 c5 第 1.3 节镜像）：出内容建议是 skill 必须给出的立场，不能只陈列行情位置/机构共识/近期叙事三段数据就丢给运营自己判断——判断责任在 skill 不在运营，运营可以否决建议，但 skill 必须先表态。

T-4 样例（MU，2026-07-22 已核可，逐字收录）：

```
🔍 MU 速查｜7/22 16:40
行情位置：$1,170 附近，52週區間上緣
機構共識：（家數+中位+分歧幅度，按當次實測返回填寫）
近期敘事：DRAM 漲價+SK 海力士會長口徑，推特 24h 9 帖全多
信號面：實盤僅 1 位無評級交易員輕倉多；CEO 5-6 月連續真賣出(S-Sale)
→ 📌 建議並入本週溫度計：情緒滿格但真金白銀沒跟上，適合跨源打架敘事
⚠️ 此為內部備忘，發佈請走 c5/c3 成稿。
```

字数：≤300 字（S-4 镜像原文"c6 内部备忘 ≤300 字"）。五段+尾行是精简数据罗列，密度优先于可读性——不需要像对外贴文那样铺陈白话解释，备忘读者是运营自己，不是新手群成员。

尾行固定文案（S-11 内部版，逐字，每份备忘必加，不可省略）：

> c6 内部备忘尾行：「⚠️ 此為內部備忘，發佈請走 c5/c3 成稿。」

## 3. 防坑镜像

spec 逐字 4 条：

> 防坑规则：
> - 非美股正股（代币化/OTC/仙股/加密）：备忘直接标"不在 bundle 覆蓋範圍"，并提示运营不要在社群硬接这类话题。
> - 共识目标价必须带家数和分歧幅度——单均值误导性大（公开研究页的常见毛病），这条规则同样约束所有对外模块。
> - 同一标的当日已查过（对话内可见）→ 复用，不重复烧额度。
> - 备忘内容是运营视角草稿，**直接复制进群等于跳过风格与免责机制，skill 在备忘尾行固定提示"此為內部備忘，發佈請走 c5/c3 成稿"**。

逐条展开：

- **非美股正股短路模板**：ticker 若是代币化股票（参照 c1 实测命中的代币化噪音 SKHYx、LAB 一类）、OTC 柜台股、仙股、或加密资产，本 skill 判定"不在 bundle 覆蓋範圍"，不跑步骤 2-3、直接短路省额度，备忘输出示范格式（非实测数据，实现时按实际 ticker 替换）：

  ```
  🔍 <TICKER> 速查
  不在 bundle 覆蓋範圍（代幣化股票/OTC/仙股/加密資產）
  提示：不建議在社群直接接這類話題，新手社群以美股正股為主。
  ⚠️ 此為內部備忘，發佈請走 c5/c3 成稿。
  ```

  判定依据同 c1 的"美股正股白名单"原则（见 c1 第 4 节防坑镜像）；即使运营对这类标的很感兴趣，备忘也只给判定结果加提示，不继续往下拉数据。

- **共识必带家数+分歧幅度**：机构共识段落必须写"X 家机构，中位 $Y，区间 $A–$B"这种完整格式，禁止简化成"目标价 $Y"这种单一均值——公开研究页的常见毛病是：单均值会让新手误以为这是一个精确数字而非一个有分歧的区间估计。这条规则同样约束 bundle 内所有对外模块（c3 機構怎麼看段落是对外版镜像，见第 2 节），c6 只是率先在内部备忘里执行。

- **当日已查复用**：同一标的当天已经速查过（本次对话内可见历史产出），直接复用那次结果，不重新烧额度重复调用步骤 1-3——除非运营明确要求"重新查一次"（比如行情剧烈波动后想拿最新数据）。

- **直接复制进群警告**：备忘的语言风格、数据密度、格式都是给运营自己看的草稿——没有白话翻译、没有名词卡、没有对外版免责声明，直接复制进群等于让新手群众看到一段没有任何合规与教育包装的裸数据。因此每份备忘尾行固定提示"此為內部備忘，發佈請走 c5/c3 成稿"（第 2 节已收录），提醒运营：真要发布，回头走 c5（速报/财报速读）或 c3（研究笔记）走一遍完整的对外产出流程，不能跳过风格与免责机制直接搬运。

补充镜像（信号面钻取时适用，仅步骤 3 触发才需要）：

- **kol_call 裂行去重（N-5）**：

  > N-5：kol_call 原帖按提及 fanout 成多行（同 URL 不同 symbol/方向）；按 source_url 去重、symbol 字段归属（实测 2026-07-22）。

  步骤 3 若命中喊单类数据，同一条原始推文可能因提及多个标的被拆成多行，统计喊单方向/热度前必须先按 source_url 去重、再按 symbol 字段归属，做法同 c4 第 3 节。

- **insider/congress 客户端过滤 transactionDate（N-6）**：

  > N-6：insider/congress 行无视 time_range（7d 返回 2020 年记录）；客户端按 transactionDate 过滤强制（实测 2026-07-22）。

  步骤 3 若命中内部人数据，必须客户端按 transactionDate 过滤（不能相信 time_range 参数已经帮忙筛过），且只认 S-Sale（卖出）、P-Purchase（买入）两种类型，F-InKind/M-Exempt 剔除——做法同 c1/c4 对应条目。

## 4. 发前自检 + 额度哨兵

c6 是内部工具，S-12 对外发布向的自检清单（繁体/大白话/名词卡/免责/多空平衡/互动钩子等对应项）不适用——本备忘没有贴文骨架、没有白话翻译、没有名词卡、没有对外版免责声明、没有互动钩子，套用整份 S-12 清单文不对题。仍然必须逐项过的只有两项：

- **数据可回溯✓**：行情位置、机构共识（家数/中位/分歧幅度）、近期叙事若提到具体涨跌幅，都必须能指向本次调用（步骤 1-3）返回值，不能凭记忆或旧查询结果补数字——呼应 S-7 铁律 2（原文：当前价/涨跌幅只引本次 MCP 返回值；新闻与 KOL 转述的百分比是二手，必经快照核实）。
- **简繁纪律✓**：备忘产出本身（T-4 模板与样例、非美股正股短路模板）用繁体中文，与 S-1 对外语言规则一致；本 skill 文件里的说明文字、判据展开一律用简体书写，两者不得混排。

额度哨兵（内部工具同样要检查，设计文档对应章节）：

> **额度哨兵**：每次模块跑完读一次 `meta.quota`（每个返回都带，实测），`remaining/limit < 15%` 时在产出末尾附一行内部提醒（不进对外贴文）："⚠️ 本月額度剩 N 次，按當前節奏約可跑 X 天，考慮降頻或升級"。额度见底导致日更断档比任何单篇内容缺失都伤运营。

c6 本身产出已经是纯内部备忘，不存在"内部/对外"的区分问题——命中阈值时直接在 S-11 内部版尾行下方再加一行额度提醒即可，两行内部提示前后叠加，互不冲突。
