# Followin MCP — Skills Community（社群运营 Bundle）

> 基于 **Followin MCP** 的 6 个 Claude Commands 兼容模块。新项目优先安装仓库根目录的标准 Agent Skill：[`skills/followin-us-equity-ops/`](../skills/followin-us-equity-ops/)。

这是运营人员的操作手册：定位声明、六模块速查表、每周运营节奏建议、安装方式、额度预算、可直接贴群的置顶帖模板、以及 Claude Code 定时任务示例，都在本文件里。

---

## 1️⃣ 定位声明

本 bundle **仅供社群运营人员使用**：运营触发 skill、审核产出、决定是否发布；群成员只消费运营发布出去的内容，不与任何 skill 产生直接或间接交互。

**不做答疑服务**（v1 明确边界）：运营代群成员现查现答某个具体问题，属于答疑服务形态，v1 版本不支持——这类需求属于 v2 backlog，见 [`docs/设计文档.md`](../docs/设计文档.md) §8「群问答回复体模式」。

六个模块的产出物性质也不相同：**c1-c5** 是"运营零加工即发"的社群贴文；**c6** 是"运营自己看"的内部备忘，产出不对外，禁止直接复制进群（每份备忘尾行都固定提示这一点）。

## 4️⃣ 安装方式

三种安装方式（路径为 `skills-community/`）：

### 方法 1：复制到全局 commands 目录（推荐）

```bash
cp skills-community/*.md ~/.claude/commands/
```

适用：运营的 Claude Code / Claude Desktop 环境里，所有项目都能触发这套 Skill。

### 方法 2：复制到项目 commands 目录

```bash
cp skills-community/*.md /path/to/your-project/.claude/commands/
```

适用：仅当前项目使用。

### 方法 3：直接 fork / clone 本仓库使用

```bash
git clone https://github.com/Followin-AI/Followin-Skills.git
cd Followin-Skills
cp skills-community/*.md ~/.claude/commands/
```

### MCP 接入

六个兼容模块合计只依赖 Followin MCP 的 3 个工具——`metrics` / `news` / `signal`。标准 Agent Skill 还会在命名账户、原始推文和信号订阅场景使用 `twitter` / `subscription`。注册账户、获取 API Key和接入方式见 [Followin MCP](https://followin.io/en/mcp)。

## 5️⃣ 额度预算

实测下来，6 个模块正常运营节奏下，全套月耗约 **600-750 点**：

| 明细项 | 单次额度(约，同 2️⃣) | 出现频率 | 额度消耗节奏(约) |
|---|---|---|---|
| c1 晨報 | ≈7 | 每日 1 次 | 7-10/天 |
| c1 開盤前瞻 | 2-3 | 每日 1 次 | ~3/天 |
| c2 週報 + c3 研報熱議榜 + c4 溫度計 | c2≈3 ／ c3≈4-6 ／ c4≈4-6 | 每周各 1 次 | ~15/周 |
| c5 熱點掃描＋速報(含財報速讀子型) | 扫描≈1 ＋ 成稿≈1-2（財報速讀单独≈1-2） | 日均 2 次 | ~120/月 |
| c6 標的速查 | ≈2 | 日均 3 次 | ~180/月 |

如实说明：
- **Basic**（1,000 点/月）可以跑，但余量收窄——扣掉 600-750 后每月只剩 250-400 点弹性，遇到财报季速查触发变频繁、或临时多加跑几次温度计/速报，容易紧张。
- **财报季**：c6 速查与 c5 财报速读的触发频率通常会明显上升，建议升级 **Pro**（5,000 点/月）留出余量——这是如实的用量说明，不是营销话术。
- **数组参数已恢复**：2026-07-23 在线回归确认 `keywords/categories/sources` 数组可按当前 schema 正常调用；`metrics` / `signal` 单次最多 5 个 keywords。以上额度是历史样本，只用于粗略预算，正式运营应以当次 `meta.quota` 为准。

## 6️⃣ 置頂帖静态模板

繁体纯文本，可直接复制贴进社群置顶：

```
📌 社群公告｜內容時刻表與免責聲明

這裡是美股新手交易社群，所有內容都是把公開研報、新聞與市場數據整理成白話文分享給大家，幫助大家更容易看懂市場現在發生什麼事，不是投資建議，也不代表任何人的操作立場。

📅 內容時刻表
🌅 每天早上：早報——整理昨夜美股怎麼收、發生了哪些大事、今天要看什麼
🌆 每天晚上（開盤前）：開盤前瞻——今晚開盤前的重點關注與盤前異動
🔍 盤中不定時：速報——遇到重大消息或財報公布，隨時補發整理，財報季會比較密集
🗓️ 每週日：一次奉上三份週更內容——週報（上週回顧＋本週看點）、本週研報熱點、本週熱議溫度計

⚠️ 以上整理自公開研報與市場數據，僅做資訊分享，不構成投資建議。
本社群所有內容均為資訊分享，不構成投資建議；投資有風險，決策請獨立判斷。
```

## 7️⃣ Claude Code 定时任务示例（可选）

若运营使用 Claude Code 作为触发端，可以用其内建的计划任务功能，把 c1 两个高频模式设成固定时间自动触发——例如：台北时间每天 08:30 触发「跑早報」（c1 晨報模式）、21:00 触发「跑開盤前瞻」（c1 開盤前瞻模式）。

具体的配置语法（计划任务命令、时间字段格式等）请以 Claude Code 当前版本的官方文档为准，本文不列出具体配置命令，避免与实际版本行为不一致。c2-c6 沿用事件驱动/周期性触发的天然节奏，不依赖定时能力；使用 Cursor / Claude Desktop / Windsurf 等无内建定时功能的客户端时，按 3️⃣ 節奏表的时段手动触发即可。
