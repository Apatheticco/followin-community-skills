## 关于 c2 週報：本轮没有样张

诚实说明，不用示意稿充数：

1. **回顾段无素材**：週報上半场要核对「上周预告的事件兑现了吗」，首次跑没有上周预告可对照，这一段结构上就跑不出来——要等连续跑满两周才有完整形态。
2. **预告段的数据源有硬伤**：实测发现财报日历会漏掉当天最重要的美股（7/23 当天 30 条返回全是印度/欧洲/OTC 小票，而同一次调用的个股字段明确显示 AAL 当天发财报，日历里却没有它）。基于这样的名单直接出週報，会给社群一份看似完整、实则漏掉大票的预告。

补救办法已写进 skill（提高 limit、剔除优先股、用个股字段交叉验证），但**在补救效果被实测确认之前，不放这个模块的样张**。

## 这批样张暴露的真实约束

样张之外，实跑还推翻了若干原本写在文档里的假设。这些修正都已回写到 [`.claude/references/followin-mcp-caveats.md`](../.claude/references/followin-mcp-caveats.md)：

| 发现 | 影响 |
|---|---|
| 异动榜实际不返回市值字段 | 需二次批量快照补市值才能过滤，额度 1→2 |
| 财报当晚基本面接口仍停留在上一季 | 当晚「实际 vs 预期」必须从新闻层取，否则会写错一整季 |
| 查询串里的普通英文词会被当股票代码 | `"earnings beat miss"` 中的 beat 被解析成仙股 BEAT，污染快照 |
| 研报榜排名基于提及数，不等于有专题报告 | GOOGL 排第 2、66 篇提及，钻取后专题报告为 0 |
| 财报日历漏掉当天美股大票 | 需交叉验证，且名单不全时必须在贴文里承认 |
| 指数批量查询产生重复行 | `^VIX` 返回两条相同行，不去重会把同一指数写两次 |

**这些不是文档推断，是实跑撞出来的。** 一个模块能不能用，看的是它撞到脏数据时会不会诚实认账——上面 c1 样张 1-2 和 c4 样张 4-2 就是两个例子。
